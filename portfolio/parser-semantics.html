Парсер векторной близости (Python)

Внутренний инструмент для ускорения ссылочной стратегии: рассчитывает релевантность доноров к целевым страницам проекта, подсказывает безопасные анкоры и формирует прозрачный XLSX-отчёт для размещений.

← Назад в портфолио

Задача

Сократить время на отбор площадок для ссылочного продвижения и снизить риски переоптимизации. Нужен инструмент, который автоматически оценит тематическую близость донора к целевой странице и учтёт качество сайта.

Что делает инструмент

  • Краулинг доноров: ограничение глубины/страниц, таймауты, троттлинг, кэш; парсинг title/h1/текста.
  • Нормализация текста: очистка HTML, токенизация, лемматизация ru/en, стоп-слова.
  • Оценка релевантности: быстрый контур TF-IDF/BM25 + точный контур эмбеддингов (SentenceTransformers) → косинусная близость.
  • Скоринг: sim_score = α·cos + β·BM25 + γ·title_match (веса задаются в конфиге).
  • Качество площадок: индексируемость, код ответа, возраст (WebArchive), заспамленность, видимость (можно подмешивать внешние метрики).
  • Анкор-помощник: предложения по анкорам (бренд/URL/безанкор/НЧ), контроль долей в портфеле.
  • Экспорт: CSV/XLSX с колонками host, url, target_url, sim_score, ix_quality, spam, indexed, age, anchor_suggestion, status.

Типовой пайплайн

  1. Загружаем целевые URL/кластеры (JSON/YAML) по проекту.
  2. Краулим кандидатов-доноров из списков/каталогов/выгрузок.
  3. Считаем sim_score, фильтруем по порогам качества.
  4. Формируем шорт-лист с рекомендованными анкорами.
  5. Ведём статусы размещений и индексирования в едином файле.

CLI-пример

python link_vsim.py \
  --target "https://antiperevozchik.ru/" \
  --donors donors.txt \
  --clusters clusters.json \
  --bm25 --embeddings all-MiniLM-L6-v2 \
  --max-pages 20 --depth 1 --timeout 10 --sleep 0.5 \
  --export out/antiperevozchik_links.xlsx --verbose

Результаты внедрения

  • Сокращение времени прескоринга доноров в 3–5 раз.
  • Рост процента релевантных размещений и одобрений вебмастерами.
  • Стабильный баланс анкоров и снижение риска переоптимизации.
  • Прозрачный отчёт для клиента: почему именно этот донор и какая ожидаемая польза.
